# 무한매수법 V4.0 전략 스펙

> 현재 구현(V1.0) 참고: `src/lib/trading/strategy.ts`

## 핵심 개념

### T값 (Turn 카운터)

매수/매도 횟수를 추적하는 소수점 카운터.

| 이벤트 | T 변화 |
|--------|--------|
| 1배 매수 | `T += 1` |
| 0.5배 매수 | `T += 0.5` |
| 쿼터손절 | `T = T × 0.75` |
| 전량매도 (사이클 종료) | `T = 0` |

### 별지점 (Star Point)

그날의 목표가. T에 따라 동적으로 변한다.

```
별% = 15 - (0.75 × T)      // TQQQ 40분할 기준
별지점 = avgCost × (1 + 별% / 100)
```

예시 (40분할 TQQQ):
| T | 별% | 별지점 (avgCost=$100 기준) |
|---|-----|--------------------------|
| 0 | 15% | $115.00 |
| 5 | 11.25% | $111.25 |
| 10 | 7.5% | $107.50 |
| 20 | 0% | $100.00 |

> 별% 공식은 종목·분할수에 따라 다를 수 있음. TQQQ 40분할 기준값 사용.

### 매수 금액 (동적 계산)

```
1회 매수금 = 잔금 / (분할수 - T)
```

V1.0의 고정 `totalCapital / rounds`와 달리, 매번 재계산.

### 전반전 / 후반전

```
전반전: T < rounds / 2
후반전: T >= rounds / 2
```

## 매일 실행 로직

### 전반전 (T < splits/2)

매일 아래 LOC 주문들을 **동시에** 제출:

1. **별지점 LOC 0.5배**: `매수금 × 0.5` @ `별지점` 가격
2. **평단가 LOC 0.5배**: `매수금 × 0.5` @ `avgCost` 가격
3. (선택) 추가 하락 대비 LOC 주문들

- 당일 장 종료 시 미체결 주문은 자동 취소됨
- 체결된 주문 기준으로만 T값과 상태 업데이트

### 후반전 (T >= splits/2)

매일 아래 LOC 주문들을 **동시에** 제출:

1. **별지점 LOC 1배**: `매수금 × 1` @ `별지점` 가격
2. (선택) 추가 하락 대비 LOC 주문들

## 매도 로직

### 익절 (목표가 도달)

별지점 이상일 때 **두 가지 주문 동시 제출**:

1. **쿼터손절 LOC @ 별지점**: 보유 수량의 25% @ `별지점`
2. **잔여 지정가 @ avgCost × 1.15**: 나머지 75% @ `avgCost × 1.15`

### 쿼터손절 (라운드 소진 시)

- 보유 수량의 25% 매도
- T값 × 0.75로 감소

## V1.0 vs V4.0 비교

| 항목 | V1.0 (현재) | V4.0 |
|------|-------------|------|
| 매수금 | 고정 `capital/rounds` | 동적 `잔금/(분할수-T)` |
| 목표가 | 고정 `avgCost×(1+targetReturn)` | 동적 `avgCost×(1+별%)` |
| T 카운터 | `roundsUsed` (정수 유사) | `T` (소수점, 쿼터 시 ×0.75) |
| 하루 주문 수 | 1개 | 2-4개 동시 |
| 매수 기준 | 평단가 이하/이상 비교 | 전반전/후반전 구분 |
| 익절 방식 | 전량 매도 | 쿼터 LOC + 잔여 지정가 |

## 구현 시 고려사항

### 아키텍처 변경 필요

현재(`reservation/route.ts`)는 **하루 1개 주문** 기반. V4.0은 **하루 N개 주문 동시 제출** + **체결 결과 수집** 필요.

```
현재 흐름:
  가격 조회 → 판단 → 주문 1개 → 상태 업데이트

V4.0 흐름:
  가격 조회 → 판단 → 주문 N개 동시 제출 (오전 cron)
                  → 체결 확인 (장 종료 후 cron)
                  → 실체결 기준 상태 업데이트
```

### 추가 필요 기능

1. **주문 목록 저장**: 당일 제출한 주문 ID 목록 Redis 저장
2. **체결 조회 API**: KIS `/uapi/overseas-stock/v1/trading/inquire-ccnl` 또는 주문 상태 조회
3. **2차 cron**: 장 종료 후(UTC 21:30 이후) 체결 결과 수집 및 상태 반영
4. **부분 체결 처리**: 일부 주문만 체결된 경우 T값 부분 반영

### KIS API

- LOC 예약주문: `POST /uapi/overseas-stock/v1/trading/order-resv`
  - TR ID: `TTTT3014U` (실전) / `VTTT3014U` (모의)
  - 파라미터: `FT_ORD_QTY`, `FT_ORD_UNPR3`
- 주문 체결 조회: `/uapi/overseas-stock/v1/trading/inquire-ccnl`
