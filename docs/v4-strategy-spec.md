# 무한매수법 V4.0 전략 스펙

> 현재 구현(V1.0) 참고: `src/lib/trading/strategy.ts`

---

## 핵심 개념

### T값 (Turn 카운터)

매수/매도 횟수를 추적하는 소수점 카운터.

| 이벤트 | T 변화 |
|--------|--------|
| 1배 매수 | `T += 1` |
| 0.5배 매수 | `T += 0.5` |
| 쿼터손절 | `T = T × 0.75` |
| 전량매도 (사이클 종료) | `T = 0` |

### 별지점 (Star Point)

그날의 목표가. T에 따라 동적으로 변한다.

```
별% = 15 - (0.75 × T)      // TQQQ 40분할 기준
별지점 = avgCost × (1 + 별% / 100)
```

예시 (40분할 TQQQ):
| T | 별% | 별지점 (avgCost=$100 기준) |
|---|-----|--------------------------|
| 0 | 15% | $115.00 |
| 5 | 11.25% | $111.25 |
| 10 | 7.5% | $107.50 |
| 20 | 0% | $100.00 |

> 별% 공식은 종목·분할수에 따라 다를 수 있음. TQQQ 40분할 기준값 사용.

### 매수 금액 (동적 계산)

```
1회 매수금 = 잔금 / (분할수 - T)
```

V1.0의 고정 `totalCapital / rounds`와 달리, 매번 재계산.

### 전반전 / 후반전

```
전반전: T < rounds / 2
후반전: T >= rounds / 2
```

## 매일 실행 로직

### 전반전 (T < splits/2)

매일 아래 LOC 주문들을 **동시에** 제출:

1. **별지점 LOC 0.5배**: `매수금 × 0.5` @ `별지점` 가격
2. **평단가 LOC 0.5배**: `매수금 × 0.5` @ `avgCost` 가격
3. (선택) 추가 하락 대비 LOC 주문들

- 당일 장 종료 시 미체결 주문은 자동 취소됨
- 체결된 주문 기준으로만 T값과 상태 업데이트

### 후반전 (T >= splits/2)

매일 아래 LOC 주문들을 **동시에** 제출:

1. **별지점 LOC 1배**: `매수금 × 1` @ `별지점` 가격
2. (선택) 추가 하락 대비 LOC 주문들

## 매도 로직

### 익절 (목표가 도달)

별지점 이상일 때 **두 가지 주문 동시 제출**:

1. **쿼터손절 LOC @ 별지점**: 보유 수량의 25% @ `별지점`
2. **잔여 지정가 @ avgCost × 1.15**: 나머지 75% @ `avgCost × 1.15`

### 쿼터손절 (라운드 소진 시)

- 보유 수량의 25% 매도
- T값 × 0.75로 감소

## 리버스모드 (Reverse Mode)

### 발동조건

| 분할수 | 발동 조건 |
|--------|-----------|
| 20분할 | T > 19 |
| 40분할 | T > 39 |

- 남은 잔금이 있어도 리버스모드로 전환. 잔금도 리버스모드에서 그대로 사용.

---

### 매도 과정

**첫날 (처음매도)**
- MOC(시장가) 무조건 매도
- 매도 개수: `floor(전체보유개수 / 등분수)`
  - 20분할: 전체개수 ÷ 10 (10등분)
  - 40분할: 전체개수 ÷ 20 (20등분)

**둘째날 이후**
- 별지점(직전 5거래일 종가 평균) 위에서 LOC 매도
- 매도 개수: `floor(직전보유개수 / 등분수)` — 매일 재계산
  - 예: 190개 → `floor(190/20)` = 9개, 181개 → 9개, 172개 → 8개

---

### 매수 과정 (쿼터매수)

- **첫날**: 매수 없음 (매도만)
- **둘째날부터**: `잔금 / 4` 금액만큼 별지점 **아래**에서 LOC 쿼터매수
  - 잔금 = 현금잔금 + 당일 매도금 합산

예시 (40분할, 원금 $20,000, 소진 시 잔금 $400):
```
첫날: MOC 매도 → 매도금 $300 발생
      총 잔금 = $400 + $300 = $700
둘째날: $700 / 4 = $175 쿼터매수 시도
셋째날: 총 잔금 재계산 → 잔금/4 매수시도
```

---

### 별지점 정의 (리버스모드)

> 일반모드의 별지점(`avgCost × (1 + 별%)`)과 **다름**

```
별지점 = 직전 5거래일 종가의 평균 (5-day MA)
```

- 오늘이 월요일 본장 직전이면 → 전주 월~금 종가 평균
- 첫날은 별지점 없이 MOC 사용
- 둘째날부터 별지점 기준으로 매수(아래)/매도(위) 구분

---

### T값 정의 (리버스모드)

| 이벤트 | 20분할 | 40분할 |
|--------|--------|--------|
| 매도 시 | `T = 직전T × 0.9` | `T = 직전T × 0.95` |
| 매수(쿼터매수) 시 | `T = 직전T + (20 - 직전T) × 0.25` | `T = 직전T + (40 - 직전T) × 0.25` |

예시 (40분할, 소진 시 T=39.5):
```
첫날 MOC 매도 → T = 39.5 × 0.95 = 37.525
둘째날 쿼터매수 → T = 37.525 + (40 - 37.525) × 0.25 = 38.14375
```

---

### 종료조건

종가가 보유평단 기준 아래 손절선을 **초과**하면 리버스모드 종료, 다음날부터 일반모드 재시작.

| 종목 | 종료 조건 |
|------|-----------|
| TQQQ | 종가 > avgCost × (1 - 0.15) |
| SOXL | 종가 > avgCost × (1 - 0.20) |

- 단 하루만에 종료될 수도 있음
- 리버스모드 동안은 종가 < 평단 상태 → 모든 거래가 손절 구간

**일반모드 복귀 시**:
- T값: 리버스모드에서 쓰던 T 그대로 유지
- 1회 매수금: `잔금 / (분할수 - T)`
- 복귀 후 또 소진(T가 1회분 미만)되면 리버스모드 재발동

### 기타

- 외부 자금 추가투입 없음 — 원금 내에서만 운용
- 규칙을 지키지 않으면 무한매수법이 아님

## V1.0 vs V4.0 비교

| 항목 | V1.0 (현재) | V4.0 |
|------|-------------|------|
| 매수금 | 고정 `capital/rounds` | 동적 `잔금/(분할수-T)` |
| 목표가 | 고정 `avgCost×(1+targetReturn)` | 동적 `avgCost×(1+별%)` |
| T 카운터 | `roundsUsed` (정수 유사) | `T` (소수점, 쿼터 시 ×0.75) |
| 하루 주문 수 | 1개 | 2-4개 동시 |
| 매수 기준 | 평단가 이하/이상 비교 | 전반전/후반전 구분 |
| 익절 방식 | 전량 매도 | 쿼터 LOC + 잔여 지정가 |

## 구현 시 고려사항

### 아키텍처 변경 필요

현재(`reservation/route.ts`)는 **하루 1개 주문** 기반. V4.0은 **하루 N개 주문 동시 제출** + **체결 결과 수집** 필요.

```
현재 흐름:
  가격 조회 → 판단 → 주문 1개 → 상태 업데이트

V4.0 흐름:
  가격 조회 → 판단 → 주문 N개 동시 제출 (오전 cron)
                  → 체결 확인 (장 종료 후 cron)
                  → 실체결 기준 상태 업데이트
```

### 추가 필요 기능

1. **주문 목록 저장**: 당일 제출한 주문 ID 목록 Redis 저장
2. **체결 조회 API**: KIS `/uapi/overseas-stock/v1/trading/inquire-ccnl` 또는 주문 상태 조회
3. **2차 cron**: 장 종료 후(UTC 21:30 이후) 체결 결과 수집 및 상태 반영
4. **부분 체결 처리**: 일부 주문만 체결된 경우 T값 부분 반영

### KIS API

- LOC 예약주문: `POST /uapi/overseas-stock/v1/trading/order-resv`
  - TR ID: `TTTT3014U` (실전) / `VTTT3014U` (모의)
  - 파라미터: `FT_ORD_QTY`, `FT_ORD_UNPR3`
- 주문 체결 조회: `/uapi/overseas-stock/v1/trading/inquire-ccnl`
